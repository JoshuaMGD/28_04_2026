# Actividad de GitHub Classroom: Propuesta de Práctica Temática Pequeña

## Descripción general
En esta actividad vas a diseñar una propuesta técnica pequeña y viable para resolver una necesidad concreta en consola o línea de comandos. El enfoque principal es **documentar bien la idea**, justificar decisiones y organizar un repositorio limpio, antes de escribir código más amplio.

## Objetivo de aprendizaje
Al terminar esta actividad, serás capaz de:
- Definir un problema técnico acotado y convertirlo en una propuesta implementable.
- Justificar la elección de lenguaje según alcance, complejidad y mantenibilidad.
- Estructurar un repositorio profesional para prácticas de programación de sistemas.
- Diseñar pruebas mínimas sin depender de frameworks externos.
- Implementar, de forma opcional, un prototipo pequeño alineado a la documentación.

## Lenguajes permitidos
Puedes elegir uno de los siguientes lenguajes como base principal:
- ARM64 Assembly
- C
- Python
- Bash

## Reglas para mantener el proyecto pequeño
1. El proyecto debe resolver **una sola necesidad principal**.
2. Máximo recomendado de 1 a 3 funcionalidades centrales.
3. Evita integrar servicios externos, red o nube.
4. Evita dependencias complejas o instalaciones pesadas.
5. Mantén funciones pequeñas y responsabilidades claras.
6. Si eliges ARM64 Assembly, limita el alcance a ejercicios muy pequeños.

## Ejemplos de temas posibles
- Mini Toolkit en ARM64
- Asistente de Estudio en Terminal
- Reporteador de Información del Sistema
- Organizador de Archivos
- Juego de Aprendizaje en Línea de Comandos

## Entregables esperados
1. Documento de propuesta (`docs/propuesta.md`) completo.
2. Caso de uso principal (`docs/caso_de_uso.md`) completo.
3. Descripción de estructura y orden del repositorio (`docs/estructura_repositorio.md`).
4. Plan de pruebas (`docs/plan_de_pruebas.md`) y checklist breve (`tests/test_plan.md`).
5. Script base de ejecución (`scripts/run.sh`) adaptado al lenguaje elegido.
6. Prototipo mínimo opcional en `src/main.<ext>`.

## Instrucciones para el estudiante
1. Lee este README completo.
2. Elige un tema pequeño y un lenguaje principal.
3. Llena primero `docs/propuesta.md`.
4. Luego documenta `docs/caso_de_uso.md`.
5. Revisa y ajusta la estructura en `docs/estructura_repositorio.md`.
6. Diseña tus pruebas en `docs/plan_de_pruebas.md` y `tests/test_plan.md`.
7. Opcional: implementa un prototipo mínimo en `src/main.<ext>`.
8. Asegúrate de que `scripts/run.sh` funcione para tu lenguaje.

## Criterios generales de evaluación
- Claridad y calidad de la propuesta técnica.
- Coherencia entre problema, alcance y solución planteada.
- Justificación razonada del lenguaje elegido.
- Orden y limpieza de la estructura del repositorio.
- Calidad del plan de pruebas y realismo de los casos.
- Viabilidad del proyecto en tiempo y complejidad.
- (Opcional) Correcta ejecución del prototipo mínimo.

## Nota importante
**Primero se documenta y planea; después, opcionalmente, se implementa un prototipo pequeño.**
La prioridad de esta actividad es tu capacidad de análisis, organización técnica y justificación de decisiones.
