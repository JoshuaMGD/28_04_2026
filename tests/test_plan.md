# Checklist Breve de Pruebas y Entrega

## 1) Checklist de pruebas mínimas
- [ ] Probé el flujo principal al menos una vez.
- [ ] Probé al menos un caso de entrada inválida.
- [ ] Verifiqué que los mensajes de error sean claros.
- [ ] Confirmé que el resultado final coincide con lo esperado.

## 2) Checklist de documentación
- [ ] Completé `docs/propuesta.md` en todas sus secciones.
- [ ] Completé `docs/caso_de_uso.md` con flujo principal y alternativo.
- [ ] Completé `docs/plan_de_pruebas.md` con tabla de casos.
- [ ] La documentación y el código son coherentes entre sí.

## 3) Checklist de ejecución
- [ ] `scripts/run.sh` se ejecuta sin usar red.
- [ ] El archivo principal en `src/` existe y funciona.
- [ ] No agregué dependencias complejas, frameworks ni servicios externos.

## 4) Checklist de entrega final
- [ ] El repositorio mantiene la estructura solicitada.
- [ ] El proyecto sigue siendo pequeño y viable.
- [ ] No hay archivos basura o temporales.
- [ ] Todo está redactado en español mexicano.
